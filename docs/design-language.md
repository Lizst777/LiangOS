# LiangOS Design Language

> Personal Digital Space — 一个连续存在的个人数字空间。

---

## 1. 项目定位

LiangOS **不是**：

- 后台管理系统
- 工具导航站
- 假操作系统
- SaaS 模板
- 仪表盘

LiangOS **是**：

- 一个个人数字空间（Personal Digital Space）
- 始终存在、连续、安静
- 属于一个人，而不是一个组织
- 氛围优先，功能服务于氛围
- 深色主导，弱光环境友好

---

## 2. UI 原则

### 2.1 空间感（Spatial）

- 页面不是卡片，而是空间中的一层
- 元素之间用呼吸感区分层级，而不是用边框
- 背景是连续的，不割裂

### 2.2 玻璃感（Glass）

- `backdrop-filter: blur(28px–30px)` 作为表层介质
- 表面使用低透明度 `rgba`（0.06–0.36），不遮盖下层背景
- 不使用纯白实心卡片

### 2.3 克制的边界

- 优先用间距和模糊区分区域，而不是边框
- 使用边框时：宽度 ≤ 1px，透明度 ≤ 0.14
- 禁用硬阴影，优先使用 `0 1px 2px rgba(…)` 级超软阴影

### 2.4 字体

- 系统字体栈，不加自定义字体
- 字重：正文 480–560，标题 600–650
- 标题 `letter-spacing: -0.03em`，正文 `-0.01em`
- 字号不跳跃，使用 `clamp()` 响应式缩放

### 2.5 颜色

- 深色模式下主背景 `#09090b` 附近
- 强调色单一：蓝色系（光色 `#3b82f6`，暗色 `#60a5fa`）
- 不使用多色系统
- muted 文字保持可读但安静（AA 级对比度即可，不追求 AAA）

---

## 3. Motion 原则

### 3.1 方式

- 所有动画使用 `cubic-bezier(0.22, 1, 0.36, 1)`
- 禁用弹跳（cubic-bezier 曲线不可超过 y=1）
- 禁用缩放动画
- 禁用旋转、翻转等炫技过渡

### 3.2 时长

| 场景 | 时长 |
|---|---|
| hover 反馈 | 0.15s–0.25s |
| 侧栏展开/折叠 | 0.35s |
| 页面切换 | 0.4s–0.5s |
| ambient glow 移动 | 6s–12s（不可察觉） |

### 3.3 空间连续

- 页面切换不是"翻页"，而是"视角转移"
- 过渡元素使用透明度 + 位移的组合（不可同时位移超过 30px）
- glow 和背景光晕应缓慢漂移，不能闪烁或跳动

---

## 4. Spacing 原则

- 使用 `--space-{1..12}` 系统（4px 递增）
- 区块之间：`--space-8`（32px）起步
- 同类元素之间：`--space-3`（12px）或 `--space-4`（16px）
- 密集列表项之间：`6px` 或 `--space-2`（8px）
- 侧边栏内边距：`--space-6`（24px）左右
- 内容区左右边距：`min(max(1rem, 4vw), 3rem)`

---

## 5. 禁止出现的风格

| ❌ 禁止 | ✅ 替代 |
|---|---|
| 彩虹渐变背景 | 单色或双色低对比背景光晕 |
| 霓虹发光 | 微弱的 `rgba(59,130,246,0.12)` glow |
| 弹跳动画 | 慢出缓入（ease-out） |
| 缩放 hover | 仅背景色变化或轻微位移（≤ 2px） |
| 白色卡片 | 半透明玻璃面板 |
| 固定宽度布局 | 弹性布局 + `max-width` 约束 |
| 按钮填充色块 | 透明或半透明背景 + hover 反馈 |
| 圆角 > 28px | 控制在 14px–18px 区间 |
| 系统字体外的装饰字体 | 仅使用系统字体栈 |

## 6. 产品方向

### 6.1 核心体验

- 每个页面属于同一个空间
- 导航是空间内的移动，不是页面跳转
- 信息密度低，留白多
- 所有交互都有延迟反馈，不突兀

### 6.2 功能取舍

- 只保留与"个人数字空间"直接相关的功能
- 删除或弱化不属于核心世界观的功能
- 不新增"管理"感的功能
- 不堆假数据

### 6.3 氛围优先级

氛围 > 功能密度 > 数据完整 > 操作效率

## 7. 开发规则

### 7.1 修改规范

- 一次只修改明确指定的文件或组件
- 不随意新增功能
- 不大范围重构
- 不改变现有业务逻辑
- 优先保持一致性和简洁性

### 7.2 样式规范

- CSS 变量驱动，不硬编码颜色或间距值
- 语义化类名（BEM-like）
- 优先使用 `color-mix()` 实现透明变化，而非手动计算 rgba
- 响应式断点仅使用 `1023px` 一个阈值
- 移动端通过折叠/隐藏元素适配，不重排布局

### 7.3 组件规范

- 无状态组件优先
- Icon 使用行内 SVG，不引入 icon 库
- 过渡效果通过 CSS transition 实现，不使用 JS 动画库
- 组件不直接引用 token 值，通过 CSS 变量传递

### 7.4 合入规范

- 每次修改后检查文件一致性
- 不遗留未使用的 CSS 或组件代码
- 避免注释掉的代码块
- 颜色调整后检查光色/暗色两种模式
